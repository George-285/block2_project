## Задачи

### Задача 1. Прогнозирование посещаемости сервиса

Прогноз количества пользователей на следующие 12 месяцев на основе исторических данных из `newborns_stats.csv`.

**Инструменты:** Python, Google Colab  
**Датасет:** `newborns_stats.csv` — помесячная статистика посещаемости, январь 2015 — февраль 2025 (123 наблюдения)

**Подход:**  
Сравнивались две модели прогнозирования временных рядов:
- Holt-Winters (тройное экспоненциальное сглаживание)
- SARIMA

По результатам кросс-валидации на тестовой выборке лучшей оказалась модель Holt-Winters с MAPE = 2.90%. Март 2025 исключён из обучающей выборки как вероятно неполный.

### Задача 2. Цепочки ID

Даны две таблицы: `users (id, new_id)` и `links (id1, id2)`, описывающие историю изменений ID пользователей. Для каждого исходного ID определяется единый финальный `new_id` — минимальный ID в цепочке.

**Реализовано два подхода:**
- рекурсивный — через рекурсивный CTE с обходом графа и защитой от зацикливания через массив посещённых вершин
- нерекурсивный — через последовательные джойны неориентированного графа рёбер на заданную глубину

**Инструменты:** PostgreSQL, DBeaver

---

### Задача 3. Очистка строк

Дана таблица `names (id, pname)`, где в поле `pname` перемешаны визуально схожие кириллические и латинские символы (а/a, о/o, с/c и т.д.).

**Подход:**
- нормализация через `TRANSLATE` с таблицей замен латинских символов на кириллические
- приведение к единому регистру через `INITCAP`
- фильтрация только проблемных строк через регулярное выражение
- дедупликация через `GROUP BY` и `ROW_NUMBER()`

**Инструменты:** PostgreSQL, DBeaver

---

### Задача 4. Симметричные пары

Дана таблица `items (id, name, category)`. Найти все пары записей, у которых полностью совпадают `name` и `category`. Пары `(id1, id2)` и `(id2, id1)` считаются идентичными, пары с одинаковыми ID исключаются.

**Подход:** самоджойн таблицы с условием `a.id < b.id` — одновременно исключает пары с одинаковыми ID и симметричные дубли.

**Инструменты:** PostgreSQL, DBeaver
